Plugin build SHAPE files - geographic grid, and frame borders

GEOGRID


Builds and save to disk shape-files in latitude-longitude projection, ellipsoid WGS84.


basename_grd.shp - vector grid of latitude and longitude.

basename_brd.shp - vector calibrated frame border.

basename_tik.shp - points of frame tics.

basename_lbl.shp - point and the labels along the frame.


* basename — defined by the base of the file name.





Procedure.


1. Determine coverage posed by grid.


Enter values for minimum and maximum values of latitude and longitude in degrees and minutes.

It is permissible to enter in degrees with decimal fractions. When the value in the minutes field should be zero.


f the project contains any layers that you can copy and use their coverage options. To do this, choose layer and press "Copy" coverage.

While the minimum and maximum values of the selected layer will be converted from layer projection projection grid, translated in the form of degrees-minutes, and inserted into the appropriate fields of the form. If necessary, the values can be changed.


Minimum input values latitudes and longitudes must always be less than the maximum.
The "-" minus sign corresponds to the Western and Southern hemisphere respectively, and only in the degrees box!



2. Specify grid settings.


The grid spacing is always positive and is set in degrees and minutes.
It is permissible to enter in degrees with decimal fractions. When the value in the minutes field should be zero.


To correctly display the grid not rectangular projections, you must must specify the number of divisions of the grid lines on the bars.

The greater the number of subdivisions slows down the display, but increases the smoothness of the arc.


3. Set the parameters borders and labels.


Steps divide frame borders, ticks and labels are selected from the drop-down lists (in minutes).


Usually step choose labels equal to or a multiple of the grid.

Three sets of ticks is generated: 1) equal step frame border 2) half step frame border and 3) with the specified step.


4 Specify the base name of the output file.


"Save as" button allows you to select a location and specify basic file name part.
In the row appears only the base part of the name. The add-on file names will be done automatically upon creation.


The generated files should not exist on disk.


By default, layers are added to the current project, add to the project can be canceled.

When layers created are added to the project, to him is applied the base styles.

Files of basic styles are in the plugin directory as styles of QGIS.


5. Run.


Execute the "OK" button
Or cancel and close the dialog by pressing "Cancel"



Crossing the 180 meridian.


Automatic build procedure when crossing the 180 meridian yet fails.


You must build the 2 geographical grid for the Western and Eastern hemisphere separately.
The coverage shall be determined from the minimum longitude to 180 degrees for the grid to the Eastern hemisphere, and from (minus!) -180 degrees to maximum longitude grid of the Western hemisphere.


If you display the grids of the Western and Eastern hemisphere need to disable the display of elements grids boundaries coincide with the Meridian 180 degrees.
Files required for this basic styles are in the plugin directory as styles of QGIS. In the title they have relevant hemispheric letters _W and _E.
